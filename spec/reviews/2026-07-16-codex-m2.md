# Codex review triage — M0–M2 (2026-07-16)

An external review (Codex, `codex-cli 0.144.1`) covered every commit after the
design docs (`3a93b80..HEAD`): the format/crypto core (M1), the extension
schemas (M1), and the A2A vendoring + v1 profile validator (M2). It reported
9 high, 12 medium, 3 low.

Each finding below has a disposition: **fixed** (addressed in this commit),
**tracked → Mx** (a genuine gap whose implementation belongs to a later
milestone; a task/plan note now exists), or **rejected** (with rationale).
The guiding rule: fix defects in already-shipped M1/M2 code and tighten
schemas where cheap; do not pull later milestones' engine work forward.

## High

| # | Finding | Disposition |
|---|---|---|
| H1 | DSSE `verify()` non-strict → low-order-key/malleable-signature forgery | **fixed** — `verify_strict` (`dsse.rs`) |
| H2 | Integers past u64 arrive as rounded `f64`, aliasing distinct values and breaking lossless signing | **fixed** — `visit_f64` rejects integer-valued magnitudes past 2⁵³ (`ijson.rs`); vectors `ijson/huge-integer`, `ijson/unsafe-integer` |
| H3 | JWK/thumbprint/purpose/DSSE key identity not cryptographically bound; "valid" vector reused one key across purposes | **fixed (transport-safe part)** — DSSE now derives the keyid from the key and requires the envelope to match (`dsse.rs`); key-binding schema is keyed by purpose (one key per purpose, known purposes only); the valid vector uses distinct keys per purpose. **fixed (M6)**: `axon_pairing::key_binding::verify` checks each transported thumbprint equals the RFC 7638 thumbprint of its JWK, behind the key-binding.v1 schema gate. |
| H4 | Result manifest can show passing evidence that doesn't exist (empty arrays, dup roles, slot→evidence unresolved) | **tracked → M11** — cross-array resolution and dup-role checks are semantic and live in `axon-evidence`; JSON Schema cannot express them. Schema remains a necessary gate; noted in the manifest schema and plan. |
| H5 | Exact Part binding and contract-array uniqueness absent | **tracked → M7** — input-manifest ↔ Message-Part binding and per-field uniqueness are code checks in `axon-contract` (already in the M7 exit criteria). |
| H6 | Agent Card validator never checks `signatures` | **fixed (M3 part)** — `axon_proto::card_sig::verify_card` verifies the JWS under the EdDSA/JOSE profile (ADR-0007), cross-checked by `jws/agent-card-eddsa`. **tracked → M6**: pinning the verification key at pairing. Structural profile validation stays a separate gate; both apply to a fetched card (`profile.md`). |
| H7 | mTLS check passed if *any* alternative mentioned mTLS | **fixed** — every requirement alternative must force mTLS and the set must be non-empty (`profile.rs`); vector `agent-card-mtls-plus-anonymous`. |
| H8 | Required-extension set could be empty; no A2A-Version check | **fixed** — `ProfileConfig::new` rejects an empty set (no `Default`); `validate_a2a_version` added. |
| H9 | No Task/Artifact/response-echo validators | **tracked → M5/M11** — outbound Task/Artifact/response validation belongs with the transport and evidence layers; `validate_task`/`validate_artifact` are noted as M5 profile work. |

## Medium

| # | Finding | Disposition |
|---|---|---|
| M1 | I-JSON does not reject Unicode noncharacters | **rejected** — RFC 7493 does not forbid noncharacters in strings; rejecting them would wrongly refuse valid content. Left as-is. |
| M2 | Parser has no item/member-count limit (DoS) | **fixed** — `MAX_NODES` node budget enforced during parse (`ijson.rs`); test `rejects_too_many_nodes`. |
| M3 | Schema error could panic on `truncate` and echoed instance content | **fixed** — char-boundary-safe truncation, content-free `masked()` message, bounded error count (`schema.rs`). |
| M4 | Contract processor/resource ceilings incomplete (`named-remote` w/o processor; no CPU/mem/count) | **tracked → M7** — the full capability/ceiling model is M7/M8; `retention_request` made required now. |
| M5 | Timestamps only shape-checked; no ordering/lifetime comparison | **tracked → M7/M8** — creation<expiry and the 24 h/1 h maxima are trusted-time checks (design §8.5), code-level. |
| M6 | Contract conditionals incomplete (rev0 task_id; text charset; per-input size) | **fixed** — rev0 forbids `predecessor_digest` and `task_id`; text inputs require `charset: utf-8` and ≤1 MiB, data ≤4 MiB (`contract.v1.schema.json`); vectors added. |
| M7 | Result-manifest ordering not enforced | **tracked → M11** — bytewise ordering is enforced when the manifest is built/validated in `axon-evidence`. |
| M8 | Cannot express "visible passing evidence" requirement | **tracked → M7/M11** — contract minimum-disclosure and slot disclosure policy. |
| M9 | Evidence not restricted to in-toto/attempt profile | **fixed (media-type part)** — evidence payload types restricted to in-toto / Axon DSSE media types (`evidence-reference`, `result-manifest`). **tracked → M11**: per-attempt/task digest binding on verifier summaries. |
| M10 | Requester outcome has no signing-key purpose | **fixed** — added `KeyPurpose::RequesterOutcome` and the `requester-outcome` key-binding purpose. |
| M11 | Message validation disconnected from negotiation; no outcome/agent-message shape checks | **tracked → M5/M7** — coupling the activated-extension set to Message validation happens where the operation is handled (transport). |
| M12 | Content/URI checks superficial (`starts_with("https://")`) | **fixed (URI part)** — `is_bounded_https_uri` requires a host and rejects control/space; content size/media-type limits are enforced by the contract manifest (schema) rather than the raw Part. |
| M13 | pbjson rejects unknown fields, contradicting §18 "preserve non-critical unknown standard fields" | **fixed** — ADR-0010: `pbjson_build::ignore_unknown_fields()` preserves non-critical unknown *standard* fields; `ignore_unknown_enum_variants` deliberately off so unknown safety-critical enum values still fail closed. Axon extension objects keep reject-unknown. Verify/forward over original bytes tracked to M5. |
| M14 | Vector harnesses materialize `Value`, hiding the raw-byte boundary | **fixed (I-JSON part)** — added the raw-bytes `ijson/` vector family driven from bytes in Rust and independently in `xcheck`. A2A/JWS/in-toto/manifest/dedup cross-checkers are tracked with their milestones (M3/M11). |

M12 (URI), M14 (I-JSON), L1, L2, L3. Tracked to a milestone or ADR: H3
(pairing), H4, H5, H6, H9, M4, M5, M7, M8, M9 (binding), M11, M13, M14 (other
families). Rejected: M1.

Suite after fixes: 50 Rust tests, 38 independently cross-checked vectors,
`cargo clippy` and `cargo fmt` clean.
